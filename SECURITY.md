## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、公開のIssueではなく、以下の方法で報告してください：

### 報告方法

1. **プライベート報告** (推奨)
   - GitHubの「Security」→「Advisories」→「New draft security advisory」から報告
   - または、プロジェクトメンテナーに直接連絡

2. **報告に含めるべき情報**
   - 脆弱性の種類と説明
   - 再現手順
   - 影響範囲
   - 可能であれば、修正案

### 対応プロセス

1. **受領確認**: 48時間以内に報告を確認
2. **評価**: 脆弱性の深刻度を評価
3. **修正**: パッチの開発
4. **リリース**: セキュリティアップデートの公開
5. **公開**: 適切な時期に脆弱性情報を公開

## セキュリティのベストプラクティス

### データの取り扱い

このシステムはブラウザのLocalStorageにデータを保存します：

- **機密情報の保存禁止**: パスワードや個人情報はサンプルデータとしてのみ使用
- **本番環境での使用**: 実際の機密データを保存する場合は、適切なバックエンドシステムとの連携を検討
- **定期的なバックアップ**: LocalStorageのデータは定期的にバックアップ

### 推奨事項

1. **HTTPS使用**: 本番環境では必ずHTTPSを使用
2. **最新ブラウザ**: サポートされているブラウザの最新版を使用
3. **アクセス制限**: 必要に応じて、適切なアクセス制限を実装
4. **定期的な更新**: システムを最新バージョンに保つ

## 既知の制限事項

### LocalStorageの制限

- **容量制限**: ブラウザにより異なるが、通常5-10MB
- **ドメイン制限**: 同一ドメイン内でのみアクセス可能
- **セキュリティ**: XSS攻撃に対して脆弱な可能性

### 認証システム

現在のシステムは開発・デモ用であり：
- **シンプルな認証**: 本格的な認証システムではありません
- **本番環境**: 実際の本番環境では適切な認証システムの実装を推奨

## 連絡先

セキュリティに関する質問や懸念がある場合：
- GitHub Issues（一般的な質問）
- Security Advisory（脆弱性報告）
